# Programme Lifecycle

The programme lifecycle provides a structured approach to delivering transformational change, from initial identification of the need through to realising benefits and closing the programme.

---

## Programme Lifecycle Overview

<div class="mermaid">
flowchart LR
    A[Identifying<br/>a Programme] --> B[Defining<br/>a Programme]
    B --> C[Managing<br/>the Tranches]
    C --> D[Closing<br/>a Programme]
    C -.-> C

    classDef blue fill:#108BB9,stroke:none,color:#fff
    class A,B,C,D blue
</div>

| Phase | Purpose | Duration |
|-------|---------|----------|
| [**Identifying**](#identifying-a-programme) | Establish need and viability | 2-4 weeks |
| [**Defining**](#defining-a-programme) | Plan the programme | 4-12 weeks |
| [**Managing Tranches**](#managing-the-tranches) | Deliver the change | Months to years |
| [**Closing**](#closing-a-programme) | Transition and close | 4-8 weeks |
{:class="table table-single table-narrow"}

## Identifying a Programme

### Purpose

Confirm there is a viable programme that will achieve the strategic objectives and deliver the required benefits.

<div class="mermaid">
flowchart LR
    A[Strategic<br/>Need] --> B[Mandate]
    B --> C[Programme<br/>Brief]
    C --> D[Approval to<br/>Define]

    classDef blue fill:#108BB9,stroke:none,color:#fff
    class A,B,C,D blue
</div>

### Key Activities

| Activity | Description |
|----------|-------------|
| Appoint SRO | Identify and appoint Senior Responsible Owner |
| Confirm mandate | Obtain executive mandate for programme |
| Develop brief | Create Programme Brief |
| Outline vision | Define Vision Statement |
| Initial assessment | High-level benefits and risks |
| Approval | Decision to proceed to definition |
{:class="table table-single table-narrow"}

### Key Deliverables

| Deliverable | Description | Link |
|-------------|-------------|------|
| **Programme Mandate** | Authority to investigate | [Template](/programmes/programme_mandate) |
| **Programme Brief** | Summary of programme | [Template](/programmes/programme_brief) |
| **Vision Statement** | Description of end state | [Template](/programmes/vision_statement) |
| **Sponsoring Group Terms of Reference** | Governance structure | [Guide](/programmes/programme_governance) |
{:class="table table-single table-narrow"}

### Gate: Approval to Define

| Criteria | Evidence |
|----------|----------|
| Strategic alignment clear | Vision Statement |
| SRO appointed | Mandate signed |
| Resources for definition | Budget approved |
| Viability confirmed | Programme Brief |
{:class="table table-single table-narrow"}

## Defining a Programme

### Purpose

Establish the programme's governance, organisation, plans, and controls, providing a solid foundation for delivery.

<div class="mermaid">
flowchart LR
    A[Programme<br/>Brief] --> B[Blueprint]
    B --> C[Benefit<br/>Profiles]
    C --> D[Programme<br/>Plan]
    D --> E[Business<br/>Case]
    E --> F[Approval to<br/>Deliver]

    classDef blue fill:#108BB9,stroke:none,color:#fff
    class A,B,C,D,E,F blue
</div>

### Key Activities

| Activity | Description |
|----------|-------------|
| Design Blueprint | Define future state using POPIT model |
| Identify benefits | Create Benefit Profiles and Map |
| Plan tranches | Develop Programme Plan |
| Establish governance | Set up Programme Board and PMO |
| Develop business case | Create detailed Business Case |
| Design projects | Define constituent projects |
| Risk assessment | Identify and assess programme risks |
{:class="table table-single table-narrow"}

### Key Deliverables

| Deliverable | Description | Link |
|-------------|-------------|------|
| **Blueprint** | Future state definition (POPIT) | [Template](/programmes/blueprint) |
| **Benefit Profiles** | Individual benefit definitions | [Template](/programmes/benefit_profiles) |
| **Benefits Map** | Benefits dependency network | [Guide](/programmes/benefits_map) |
| **Programme Plan** | High-level schedule | [Template](/programmes/programme_plan) |
| **Business Case** | Investment justification | [Template](/programmes/business_case) |
| **Governance Framework** | Roles, boards, controls | [Guide](/programmes/programme_governance) |
| **Stakeholder Map** | Stakeholder analysis | [Template](/programmes/stakeholder_management) |
| **Risk Register** | Programme-level risks | [Template](/programmes/risk_register) |
| **Quality Management Strategy** | Quality approach | [Guide](/programmes/quality_strategy) |
{:class="table table-single table-narrow"}

### Gate: Approval to Deliver

| Criteria | Evidence |
|----------|----------|
| Blueprint complete | POPIT model defined |
| Benefits quantified | Benefit Profiles complete |
| Business Case positive | NPV/ROI acceptable |
| Risks acceptable | Risk Register reviewed |
| Resources identified | Resource plan in place |
| Governance established | Board and PMO in place |
{:class="table table-single table-narrow"}

## Managing the Tranches

### Purpose

Deliver the programme through coordinated tranches, each delivering a step-change in capability and benefits.

<div class="mermaid">
flowchart TD
    A[Tranche 1] --> B[Transition 1]
    B --> C[Benefits Review 1]
    C --> D[Tranche 2]
    D --> E[Transition 2]
    E --> F[Benefits Review 2]
    F --> G[Tranche n...]

    classDef blue fill:#108BB9,stroke:none,color:#fff
    class A,B,C,D,E,F,G blue
</div>

### Tranche Structure

| Element | Description |
|---------|-------------|
| **Tranche** | Group of projects delivering related capabilities |
| **Transition** | Moving to new operational state |
| **Benefits Review** | Assessing benefits realised |
{:class="table table-single table-narrow"}

### Key Activities

| Activity | Description |
|----------|-------------|
| Manage projects | Coordinate constituent projects |
| Monitor progress | Track against Programme Plan |
| Manage transitions | Embed changes into operations |
| Realise benefits | Track and measure benefits |
| Manage risks | Monitor and respond to risks |
| Stakeholder engagement | Maintain support and manage resistance |
| Update business case | Refresh forecasts |
| Governance | Regular board meetings, assurance |
{:class="table table-single table-narrow"}

### Key Deliverables

| Deliverable | Description | Link |
|-------------|-------------|------|
| **Tranche Plan** | Detailed tranche schedule | [Template](/programmes/tranche_plan) |
| **Project Briefs** | Initiation of projects | [Guide](/projects/project_brief) |
| **Highlight Reports** | Regular progress reports | [Template](/programmes/highlight_report) |
| **Transition Plans** | Operational transition | [Template](/programmes/transition_plan) |
| **Benefits Realisation Reports** | Benefits tracking | [Template](/programmes/benefits_report) |
| **End Tranche Reports** | Tranche completion | [Template](/programmes/end_tranche_report) |
| **Updated Business Case** | Refreshed justification | [Guide](/programmes/business_case) |
{:class="table table-single table-narrow"}

### Tranche Review Gate

| Criteria | Evidence |
|----------|----------|
| Tranche objectives met | End Tranche Report |
| Benefits on track | Benefits Realisation Report |
| Business case valid | Updated Business Case |
| Risks acceptable | Risk Register |
| Next tranche planned | Tranche Plan |
{:class="table table-single table-narrow"}

## Closing a Programme

### Purpose

Confirm that the programme has delivered its objectives, transition to business as usual, and formally close.

<div class="mermaid">
flowchart LR
    A[Final<br/>Transition] --> B[Benefits<br/>Confirmed]
    B --> C[Lessons<br/>Captured]
    C --> D[Programme<br/>Closed]

    classDef blue fill:#108BB9,stroke:none,color:#fff
    class A,B,C,D blue
</div>

### Key Activities

| Activity | Description |
|----------|-------------|
| Final transition | Complete handover to operations |
| Confirm benefits | Final benefits assessment |
| Lessons learned | Capture and share learning |
| Close projects | Ensure all projects closed |
| Release resources | Return resources to organisation |
| Archive documentation | Store programme records |
| Formal closure | Programme Board sign-off |
{:class="table table-single table-narrow"}

### Key Deliverables

| Deliverable | Description | Link |
|-------------|-------------|------|
| **End Programme Report** | Final programme report | [Template](/programmes/end_programme_report) |
| **Benefits Realisation Review** | Final benefits assessment | [Template](/programmes/benefits_review) |
| **Lessons Learned Report** | Captured learning | [Template](/programmes/lessons_learned) |
| **Handover Documentation** | Operational handover | [Guide](/programmes/handover) |
| **Closure Notification** | Formal closure | [Template](/programmes/closure_notification) |
{:class="table table-single table-narrow"}

### Gate: Programme Closure

| Criteria | Evidence |
|----------|----------|
| Objectives achieved | End Programme Report |
| Benefits realised/on track | Benefits Review |
| Operations stable | Handover complete |
| Lessons captured | Lessons Learned Report |
| Board approval | Closure sign-off |
{:class="table table-single table-narrow"}

## Lifecycle Comparison

### Programme vs Project Lifecycle

| Aspect | Project | Programme |
|--------|---------|-----------|
| **Duration** | Months | Years |
| **Focus** | Outputs | Outcomes |
| **Structure** | Stages | Tranches |
| **Benefits** | Enabled | Realised |
| **Change** | Controlled | Embraced |
| **Scope** | Fixed | Evolving |
{:class="table table-single table-narrow"}
